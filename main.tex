\section{Introduction}
The manipulation of pitch in digital signals is not achieved by simply 
scaling frequencies, but through structured transformations in the time-frequency 
domain that reshape signals in a variety of ways.

\subsection{Pitch Shifting}
Pitch shifting is a desirable signal processing effect for a variety of practical reasons. 
Historically, pitch manipulation emerged from research in vocal processing 
(giving rise to the term `vocoder' a concatenation of the words `vocal' and `coder') \cite{dolson1986phase}. 
Today, pitch-shifting algorithms are widely used in music production, film, 
voice acting, and media. Specifically in music production,  
digital vocal correction tools such as autotuning rely on pitch 
modification techniques to adjust a singer's intonation while preserving 
timing and vocal characteristics.

Pitch shifting is not limited to vocal signals, however, it is also commonly applied 
to instrumental audio. For example, electric guitarists use pitch-shifting 
pedals and digital audio plugins to alter tuning without physically adjusting 
string tension. This capability allows musicians to explore alternate tunings
without modifying their instrument hardware or spending vast amounts of time retuning extended ranged guitars. 
Such applications highlight the broad practical demand for high-quality, 
real-time pitch-shifting algorithms.

\subsection{The Phase Vocoder Approach}
The phase vocoder is a signal processing technique used for both time-scale and pitch modifications.
This algorithm was chosen to be the core processing algorithm for this project due to the fact that
the family of vocoder algorithms is known for their reliability, ability to handle polyphonic signals, 
and their widespread adoption many modern signal processing applications. Furthermore, the phase vocoder
is a real-time algorithm, meaning it has the capability to execute, in its entirety, in an unperceivable 
amount of time. Hence, this algorithm, along with the JUCE audio plugin framework, will provide a basis for creating 
a real-time pitch-shifting audio plugin.

\section{Background}

\subsection{Discrete Fourier Transform}
The Discrete Fourier Transform (DFT) is a fundamental tool used in digital signal processing that 
enables sophisticated analysis and synthesis of discrete-time signals \cite{lyons2010understanding}.   
The DFT transforms a signal from the time domain into its frequency-domain representation,  
allowing the signal to be expressed as a sum of sinusoids \cite{lyons2010understanding}. The DFT accepts a finite 
sequence of $N$ discrete samples and produces $N$ complex-valued outputs, each representing the signal's magnitude 
and phase at one of $N$ evenly-spaced frequency bins. 
The output of the DFT can be better understood as a sequence of $N$ complex numbers representing amplitude and phase for each analysis frequency \cite{mcfee_dstbook}.  
Mathematically, the DFT is defined as: 
\begin{align}
  X[m] = \sum_{n=0}^{N-1} x[n] \, e^{-j 2\pi n\frac{m}{N}}
\end{align}
where $N$ is the number of samples from the input signal $x[n]$, $n$ denotes the time-domain index, $m$ represents the frequency bin index, 
$n$ is the index of the input sequence in the time-domain and $j = \sqrt{-1}$. Each coefficient in $X[m]$ is complex-valued and can be 
expressed in terms of magnitude and phase, which describe the contribution of a corresponding frequency component to the overall signal \cite{mcfee_dstbook}.
Another important property of the DFT is that it is invertible; meaning a signal can be transformed from the frequency domain to the time domain. It can be described
mathematically as: 
\begin{align}
  x[n] = \frac{1}{N} \sum_{n=0}^{N-1} X[m] \, e^{j 2\pi n\frac{m}{N}}
\end{align}
Since the forward direction of the DTF breaks a signal into its fundamental sinusoids, the inverse DFT perfoms the opposite.
It computes each component of a signal $x[n]$ as the average of the $n$th samples of the DFT's fundamental sinusoids \cite{mcfee_dstbook}.
The inverse DFT is a crucial component of the phase vocoder as it allows for a signal to be transformed back to the time domain after being modified
in the frequency domain.In practice, the DFT is much too slow for modern applications. Therefore, applications today use the Fast Fourier Transform (FFT), 
an optimized algorithm for computing the DFT. A common method is the Cooley and Tukey algorithm, which requires that the input signal length be a power of two. \cite{mcfee_dstbook}. 

\subsection{Windowing}
Windowing is a function used to isolate parts of a discrete time signal so that analysis on the signal can be performed on contiguous blocks \cite{barry2008time}.
More specifically, the input signal is multiplied by a windowing function whose domain is nonzero for a particular range. This results in a small localized snapshot of the input signal
where every value outside of the window's domain is reduced to zero \cite{royer2019pitch}. Figure 1 below demonstrates the functionality of a window function.

\subsection{Short-Time Fourier Transform}
The Short-Time Fourier Transform (STFT) is a time-frequency representation used to compute the DFT of contiguous windowed segments of a signal. In practice, 
however, the STFT uses the FFT as this is more computationally efficient.
This is accomplished by multiplying the signal by a windowing function so that only a small portion of the signal is nonzero \cite{dolson1986phase}. 
Mathematically, it is similar to equation $(1)$ but with a few modifications: the addition of a windowing function, the domain of the input signal, 
and how the output is interpreted. Below is the STFT: 
\begin{align}
  X[m,k] = \sum_{n=0}^{N-1} x[n+kH]w[n]e^{-2\pi jmn/N}
\end{align}
Where $X[m, k]$ is the 2-dimensional output array indexed by both frequency bin and time frame and $x[n + kH]$ is the section of the input signal 
determined by frame index and hop size \cite{royer2019pitch}. The outcome of the STFT is a representation that transforms a signal from the time 
domain to the frequency domain. In this form, the signal's frequency components can be systematically modified and recombined to reconstruct a new signal. 


Since the STFT only computes the DFT of a particular section of a given signal, it must be shifted in order to process another section of the signal. 
The amount in which the STFT is shifted is known as the \textit{Hop Size}, denoted as $H$. The hop size is determined by how much the successive STFTs should overlap, 
more on this in Section 4.3. Typical hop sizes include $75\%$, $50\%$, $33.3\%$, and $20\%$ overlap. \cite{degotzen2000traditional}  
Lastly, just as the DFT has an inverse, so to does the STFT. However, instead of reconstructing the entire signal, the inverse STFT only reconstructs a particular
chunk of a signal. This is especially useful in reconstructing a signal after modifying it in the frequency domain. 


\section{Related Works}

\subsection{Classical Phase Vocoder Approaches}
Pitch-shifting has been approached by a number of time--frequency methods. 
Early methods, such as Dolson's (1986), employed the use of the Short-Time 
Fourier Transform in conjunction with the phase vocoder algorithm. 
In particular, Dolson describes a pitch-shifting method that relies on 
time-scale manipulation to modify the pitch of a given signal \cite{dolson1986phase}. 
For example, by stretching a signal, making it longer in the time domain, 
and then rescaling it back to its original length, the overall frequency 
content of the signal is effectively decreased, which corresponds to 
lowering the signal's pitch. Dolson acknowledges, however, that this 
approach is imperfect: specific constraints must be enforced when 
performing a pitch shift, as failing to do so may introduce unwanted 
artifacts into the signal or result in a reconstructed signal that has 
lost information \cite{dolson1986phase}. While effective for monophonic (single note) pitch scaling, 
this classical time-scale modification framework treats spectral bins independently, 
which can lead to phase incoherence and unwanted artifacts. 
These limitations motivated subsequent refinements to the phase vocoder algorithm.

\subsection{Phase Vocoder Optimizations}

Unlike the original process chain described by Dolson (1986), 
which applies a uniform scaling factor to all spectral components, 
later work by Laroche and Dolson (1999) introduces refinements that improve 
frequency coherence \cite{laroche1999new}. In particular, they highlight a new 
peak-detection paradigm, where peaks refer to how much a particular frequency 
is in the input signal. Once the peaks are detected, they can be shifted around 
by a pitch-factor and unlike the version described by Dolson (1986), the pitch-factor 
is not restricted to an integer amount \cite{laroche1999new}.
 
In addition to optimizing frequency resolution, modern implementations of the phase vocoder 
shift away from the standard two-step timescale and resample approach and 
adopt an analysis and resynthesis paradigm \cite{degotzen2000traditional}. 
The new approach leverages phase information obtained from the STFT step to more accurately 
reconstruct the modified signal \cite{ degotzen2000traditional}. More specifically, a phase adjustment 
value can be derived from the frequency bins across two frames and when added to the original frequency 
approximation the accuracy of the frequency estimation is increased \cite{lim2007phasevocoder}.


\subsection{Alternative Pitch-Shifting Methods}

The prior sections highlight the phase vocoder approach to pitch-shifting, a frequency-domain algorithm that 
converts a signal from the time domain to the frequency domain to compute the pitch shift. An alternative approach uses time-scale modification (TSM) algorithms,
 which operate entirely in the time domain \cite{rai2019analysis}. 
 A common algorithm belonging to the TSM family is the Overlap-Add (OLA) algorithm \cite{royer2019pitch}.

The idea behind these algorithms is that pitch shifting and time scaling are essentially the same; meaning if you have one, 
either a pitch shifter or time scaler, then you effectively have the other. OLA makes use of this paradigm. The algorithm first partitions the input signal into frames, much like the STFT. 
Then, if the goal is to pitch down the signal, when resynthesizing the signal it is constructed in such a way that extra space is added between the reconstructed frames, increasing the length of the signal. 
On the contrary, if the goal is to make the signal's pitch higher, then the algorithm reconstructs the signal in such a way where the space between contiguous frames is reduced, effectively shortening the signal's length \cite{royer2019pitch}.

Since TSM algorithms operate in the time domain and thus change the input signal's duration, either lengthening or shortening it, these algorithms must incorporate a resample step in order to convert the modified signal back to the original signal's duration \cite{rai2019analysis}.
Thus, these alternative algorithms are viable options; however, purely TSM-based algorithms are often outperformed by frequency-based algorithms \cite{royer2019pitch}.

\subsection{Analysis}
The first step of the phase vocoder algorithm is the analysis step. The primary goal of this step is to break an input audio signal 
into a succession of overlapping frames so that each frame is processed individually. The analysis step segments the signal into frames
because it is designed to operate in a real-time environment, such as an audio plugin. In this setting, the phase vocoder does not have access to 
the entire signal at once, as it is being continuously fed into the phase vocoder from a source. Thus, the phase vocoder is not concerned with the global scope
of the signal, it only needs to process the signal locally.  


In practice, the STFT is used to partition the audio signal into frames of length $N$.
This is accomplished by multiplying the signal frame by the Hann window function. Once the signal frame has been isolated, 
the FFT is computed on the windowed signal chunk, converting it from the time domain to the frequency domain. To process the next frame, 
the analysis window moves ahead by $H$ (hop size) number of samples.

\subsection{Synthesis}

Once the signal frame has been modified, the IFFT converts the signal back to the time domain. Then the output is reconstructed using
the Constant Overlap-Add (COLA) paradigm. A detailed explanation of COLA can be found here \cite{smith2011sasp}. The key idea is that 
if samples in the window function, offset by $H$, add to a constant $C$, then the window possesses the "Constant Overlap-Add Property" \cite{smith2011sasp}. The 
mathematical expression below demonstrates the COLA property: 
\begin{align}
  \sum_{m=-\infty}^{\infty} w(n - mH) = 1, \quad \forall n \in \mathbb{Z}
\end{align}
This project employs the Hanning window function with a $75\%$ overlap. To prove it possesses the COLA property we can graph several Hanning window functions 
with a $75\%$ overlap and their corresponding overlap value. Figure 2 shows that by graphing five Hanning window functions with a $75\%$ overlap satisfies the COLA property. Notice a $66.67$\% ($\frac{2}{3}$) overlap
